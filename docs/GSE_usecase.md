## UC-01. 회원가입 & 이메일 수집

- **액터:** 계정 사용자
- **사전조건:** 없음
- **사후조건:** `users` 테이블에 신규 row 생성. `marketing_consent` 값 저장.

**기본 흐름:**
1. 사용자가 랜딩페이지에서 예시 플립북을 확인하고 '무료로 시작하기'를 클릭한다.
2. 회원가입 페이지에서 이메일, 비밀번호, 이름, 소속 교회(선택)를 입력한다.
3. **마케팅 수신 동의 체크박스**(선택): "갓생북 은혜의 업데이트 소식을 이메일로 받겠습니다."
   - 동의 여부와 관계없이 이메일은 서비스 공지 목적으로 수집됨을 안내
4. Supabase Auth로 계정 생성. `users` 테이블에 row 생성.
   - `marketing_consent`: true / false
5. 로그인 성공 → 이벤트 생성 대시보드로 이동.

**예외 흐름:**
- 이미 가입된 이메일 → "이미 사용 중인 이메일입니다" 안내
- 이메일 형식 오류 → 실시간 유효성 검사 메시지

**관련 테이블:** `users`
**전략적 의의:** 수집된 이메일은 추후 유료 기능 출시 / 업그레이드 소식 발송에 활용.

## UC-02. 단체 이벤트 생성

- **액터:** 계정 사용자
- **사전조건:** UC-01 완료 (로그인 상태)
- **사후조건:** `events` 테이블에 신규 row 생성. QR 코드 URL 생성 및 저장.

**기본 흐름:**
1. 대시보드에서 '새 이벤트 만들기' 클릭.
2. 이벤트 유형 선택: 수련회 / 선교여행 / 캠프 / 예배 / 모임
3. 이벤트 이름, 기간, 예상 참여 인원 입력.
4. 시스템이 `events` 테이블에 저장 + 고유 QR 코드 URL 생성.
5. QR 코드가 화면에 표시됨. 리더가 참여자에게 공유.

**예외 흐름:**
- 필수 필드 누락 → 해당 필드 강조 + 입력 요청

**관련 테이블:** `events`, `users`

## UC-03. 개인 기록 이벤트 생성

- **액터:** 계정 사용자
- **사전조건:** UC-01 완료 (로그인 상태)
- **사후조건:** `events` 테이블에 신규 row 생성 (`is_group = false`, `qr_code_url = null`)

**기본 흐름:**
1. 대시보드에서 '새 이벤트 만들기' → '개인 기록' 선택.
2. 이벤트 이름, 카테고리 입력.
3. 시스템이 저장. QR 코드 없음.
4. 계정 사용자가 직접 기록 시작.

**관련 테이블:** `events`, `users`

## UC-04. QR 코드로 참여자 접속

- **액터:** 참여자
- **사전조건:** UC-02 완료. QR 코드가 참여자에게 공유된 상태.
- **사후조건:** `participants` 테이블에 신규 row 생성. `localStorage`에 `session_token` 캐싱.

**기본 흐름:**
1. 참여자가 모바일로 QR 코드 스캔.
2. 갓생북 은혜 웹앱 이벤트 접속 페이지로 이동.
3. 이름 입력 (소속/반 선택 입력).
4. 시스템이 `participants` 테이블에 row 생성. `session_token` 발급.
5. `localStorage`에 `session_token` 캐싱.
6. '나의 플립북' 대시보드로 이동. 환영 메시지 + 기록 가이드 표시.

**예외 흐름:**
- 동명이인 존재 → 소속/반 입력 필드 추가 표시
- QR 이벤트 비활성 상태 → "이벤트가 종료되었습니다" 안내
- `localStorage` 초기화 → 이름 재입력으로 기존 세션 복구

**관련 테이블:** `participants`, `events`

## UC-05. 활동 기록 입력 (직접 작성)

- **액터:** 참여자 (단체 이벤트), 계정 사용자 (개인 이벤트)
- **사전조건:** UC-04 완료 또는 로그인 상태.
- **사후조건:** `entries` 테이블에 row 생성. 사진 → Supabase Storage 저장.

**기본 흐름:**
1. '기록하기' 버튼 탭.
2. 섹션 선택 (리더가 설정한 목차 기반).
3. 기록 입력 페이지 표시:
   - **묵상 가이드 질문** (이벤트 유형 + 섹션 매칭): 예) "오늘 받은 말씀 중 한 문장을 써보세요"
   - **예시 완성글 보기** 버튼으로 참고 예시 확인 가능
   - 본문 직접 작성 (최대 500자)
   - 성경 구절 직접 입력 필드 (선택)
   - 인용 문구 직접 입력 필드 (선택)
4. 사진 첨부:
   - 카메라 촬영 또는 사진첩 업로드 (권장)
   - "📸 사진 한 장이 기억을 살려줘요!" 안내 표시
   - **사진 없을 경우**: 기본 템플릿 이미지 선택 UI (church / nature / verse 분류, 선택 필수)
5. '저장하기' 탭.
6. `entries` 테이블에 저장. `participants.record_count` +1.
7. 플립북 대시보드로 복귀. 새 페이지 실시간 반영.

**예외 흐름:**
- 본문 미입력 → '저장하기' 버튼 비활성화
- 사진 업로드 실패 → "다시 시도해주세요" 안내. 사진 없이 진행 가능 (템플릿 선택으로 대체)
- 네트워크 오류 → `onChange` 이벤트마다 Supabase upsert(debounce 1초)로 자동 임시저장

**관련 테이블:** `entries`, `participants`, `sections`

## UC-06. 플립북 감상

- **액터:** 참여자, 계정 사용자
- **사전조건:** 최소 1개의 entry 존재. 세션 또는 로그인 유효.
- **사후조건:** 없음 (읽기 전용)

**기본 흐름:**
1. '나의 플립북' 대시보드로 이동.
2. Supabase DB에서 해당 이벤트의 `entries` + `group_contents` 로드. (useSWR 캐싱)
3. `react-pageflip`이 클라이언트에서 플립북 렌더링.
   - 페이지 타입: `TitlePage` / `EntryPage` / `GroupContentPage`
   - 본문 + 성경 구절 + 인용 문구 + 사진(또는 템플릿 이미지) 레이아웃
4. 스와이프 / 넘김 버튼으로 책장 넘기기 (애니메이션 + 사운드).
5. 새 콘텐츠 추가 시 "새로운 페이지가 추가되었습니다!" 알림.

**예외 흐름:**
- entry 0개 → 빈 플립북 + 첫 기록 유도 UI
- 네트워크 오류 → PWA 캐시로 오프라인 감상 지원

**관련 테이블:** `entries`, `group_contents`, `sections`

## UC-07. 그룹 공통 콘텐츠 업로드

- **액터:** 계정 사용자
- **사전조건:** UC-02 완료 (단체 이벤트 활성화)
- **사후조건:** `group_contents` 테이블에 row 생성. 모든 참여자 플립북에 자동 반영.

**기본 흐름:**
1. 이벤트 관리 대시보드에서 '공통 콘텐츠 업로드' 탭.
2. 콘텐츠 유형 선택: `photo` / `notice` / `summary`
3. 파일 또는 텍스트 입력 후 저장.
4. Supabase Storage 저장 + `group_contents` row 생성.
5. 전체 참여자 플립북에 즉시 반영.

**예외 흐름:**
- 파일 크기 초과 → 용량 제한 안내 (사진 최대 10MB)
- 권한: `creator_id`만 업로드 가능

**관련 테이블:** `group_contents`, `events`

## UC-08. 이벤트 모니터링

- **액터:** 계정 사용자
- **사전조건:** 이벤트 1개 이상 존재
- **사후조건:** 없음 (읽기 전용)

**기본 흐름:**
1. 계정 대시보드에서 이벤트 목록 확인.
2. 이벤트 선택 → 상세 대시보드 진입.
3. 시스템이 표시:
   - 단체: 참여자 목록, 참여자별 `record_count`, 전체 기록 수
   - 개인: 본인 기록 수
4. QR 코드 재공유 버튼 제공.

**예외 흐름:**
- 참여자 0명 → QR 코드 재공유 안내

**관련 테이블:** `events`, `participants`, `entries`

## UC-09. PDF 다운로드 (무료)

- **액터:** 계정 사용자
- **사전조건:** 최소 1개의 entry 존재. PC 브라우저.
- **사후조건:** PC에 `.pdf` 파일 다운로드.

**기본 흐름:**
1. 이벤트 대시보드에서 'PDF 다운로드' 버튼 클릭.
2. 다운로드 옵션 선택: 개별 참여자 PDF / 그룹 통합 PDF.
3. PC 브라우저에서 `html2pdf.js` 클라이언트 사이드 실행.
   - Supabase DB에서 `entries`, `group_contents` 전체 조회
   - 플립북 레이아웃 HTML 구성 → PDF 변환
4. 완료 시 PC에 파일 다운로드.

**예외 흐름:**
- 모바일 접속 → "PC 브라우저에서 다운로드해 주세요" 안내
- 대용량 이미지 처리 → "생성에 시간이 소요될 수 있습니다" 안내

**관련 테이블:** `entries`, `group_contents`
